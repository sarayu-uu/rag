RAG Repository Architecture
===========================

Note:
- This document reflects the current repo structure after Phase 1, 2, and 3 backend ingestion work.
- Large generated folders such as `venv`, `frontend/node_modules`, and Python `__pycache__` folders are not expanded.
- `.env` is mentioned only as a config file; no secret values are documented here.

Current backend ingestion flow:
1. A client calls `/ingestion/load` or `/ingestion/uploadtochunk`.
2. The backend validates the input and stores uploaded files in `backend/uploads`.
3. The ingestion router selects the right loader and extracts raw text.
4. The text-cleaning layer normalizes and cleans the extracted text.
5. The chunking layer creates recursive or semantic chunks.
6. The backend stores document metadata in `documents` and chunk records in `document_chunks`.

Current Phase 1-3 outcome:
- Phase 1: FastAPI, config, MySQL, SQLAlchemy, and core schema are in place.
- Phase 2: Multi-format ingestion and document metadata storage are working.
- Phase 3: Cleaning, chunking, PDF page-aware chunks, semantic chunking, and chunk persistence are working.
